================================================================================
FRONTEND MIGRATION: STREAMLIT → REACT + VITE
================================================================================

Date: 31 March 2026
Status: Complete rewrite for maintainability, debugging, and UX

================================================================================
WHAT CHANGED
================================================================================

1. FRONTEND FRAMEWORK
   Before: Streamlit (Python-based, high-level abstraction)
   After:  React 18 + Vite (JavaScript, component-driven, industry standard)
   
   Why:
   - Streamlit abstracts too much control; harder to debug custom interactions
   - React components are composable, testable, and easier to refactor
   - Vite provides faster dev server and better build performance
   - Vanilla JS gives full control over UI details and state management

2. TECHNOLOGY STACK
   Before: Python + Streamlit + Plotly
   After:  React 18 + Vite + Vanilla CSS
   
   Why:
   - No heavy plotting library dependency; focus on data tables and forms
   - CSS Grid/Flexbox for responsive, debuggable layouts
   - Smaller bundle; faster load times

3. UI/UX REDESIGN
   Before: 4 pages (Global Pulse, Student Discovery, Diagnostic Report, Manual Entry)
   After:  Single-page dashboard with split layout
   
   Why:
   - Faster context switching (left: upload/predict; right: list/details)
   - Reduce cognitive load; clear information hierarchy
   - Real-time student selection without page navigation
   - More suitable for a focused "small college project"

4. VISUAL DESIGN
   Before: Dark theme with colorful badges, glass-morphism
   After:  Clean dark theme (same palette), glass cards, better spacing
   
   Why:
   - Better contrast and readability
   - Consistent throughout single page
   - Easier to maintain CSS
   - Mobile-responsive grid

5. COMPONENT STRUCTURE
   New files created:
   - src/main.jsx                → Entry point
   - src/App.jsx                 → Main layout manager
   - src/lib/api.js              → All API calls (same endpoints)
   - src/components/Header.jsx   → Status bar
   - src/components/StatsPanel.jsx → 4 metric cards
   - src/components/UploadPanel.jsx → CSV uploader
   - src/components/StudentsTable.jsx → Sortable student list
   - src/components/StudentDetails.jsx → SHAP diagnostics
   - src/components/PredictionForm.jsx → Manual entry form
   - src/styles.css              → Unified styling

6. SETUP & DEPLOYMENT
   Before: 
      pip install streamlit
      python run.py frontend
   
   After:
      cd frontend && npm install
      python run.py frontend
      (or: cd frontend && npm run dev)
   
   Why:
   - npm packages are the JavaScript standard
   - Clearer separation of frontend/backend build processes
   - Production: npm run build → static bundle ready for any CDN

7. API INTEGRATION
   No changes to backend API.
   Frontend still calls the same FastAPI endpoints:
   - POST /upload
   - GET /students
   - GET /student/{id}
   - POST /predict
   - GET /health
   
   This ensures zero disruption to your backend or database.

SETUP INSTRUCTIONS
================================================================================

1. Install frontend dependencies (one-time):
   cd frontend
   npm install
   cd ..

2. Start the backend as before:
   python run.py backend

3. Start the frontend (in another terminal):
   python run.py frontend
   → Opens http://localhost:5173 automatically

4. Or start both together:
   python run.py both
